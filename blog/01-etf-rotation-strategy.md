# ETF 轮动策略的量化核心——基础原理与实现方案

> 一套 72 只 ETF、15 个板块的动量轮动策略，核心打分逻辑不到 100 行。本文从数学原理出发，拆解它的设计思想与工程落地。

---

## 一、为什么做 ETF 轮动

A 股有一个显著特征：**板块轮动剧烈，行业分化严重**。同一月份，军工可能涨 15%，消费却跌 8%。传统的 buy-and-hold 沪深 300 年化收益常年在 5%~8%，最大回撤动辄 30%+。

如果能捕捉到「哪个板块正在走强」的信号，在板块之间按月轮动，理论上可以显著提升夏普比率。

这就是 **ETF 轮动策略** 的出发点。它的三个核心优势：

1. **ETF 免印花税**，交易成本远低于个股轮动；
2. **ETF 是板块的代理变量**，天然分散个股风险，不需要做协方差矩阵估计；
3. **动量效应在 ETF 层面比个股更稳定**——机构定价特征使趋势延续性更强。

这套策略的前身来自聚宽（JoinQuant）社区广为流传的 ETF 轮动框架。我把它的核心逻辑拆解后用 Go 语言完整实现，跑在自己的服务器上每天出信号。本文讲的是数学模型和设计取舍，不绑定语言。

---

## 二、核心打分公式

策略的心脏是一段不到 100 行的动量打分算法。输入是某只 ETF 最近 21 个交易日的收盘价序列，输出是一个动量得分。得分最高的 ETF 就是今天应该持有的标的。

整个计算分为四步：

### 2.1 取对数价格

第一步，将收盘价转为自然对数：

```
y_i = ln(close_i),   i = 0, 1, ..., 20
```

为什么取对数？对数价格的时间序列具有可加性——连续复利下，`ln(P_t / P_0)` 等于各期对数收益率之和。对一条对数价格序列做线性回归，斜率就是日均对数收益率。

### 2.2 设计递减权重

```
w_i = 1.0 + (i / 20),   i = 0, 1, ..., 20
```

权重从 1.0 线性增长到 2.0（等间距 1/19），越接近今天的数据权重越大。

为什么不是等权？假设一只 ETF 在 10 天前暴涨但最近 5 天阴跌——等权回归会让它仍拿到高分，但趋势交易员显然不会买它。加权回归让近期数据的拟合误差代价更高，对趋势拐点更敏感。

### 2.3 加权线性回归

对 (x_i = i, y_i = ln(close_i)) 做加权最小二乘（WLS），求解 `y = a·x + b`：

```
斜率 a = (Σw · Σwxy − Σwx · Σwy) / (Σw · Σwxx − Σwx²)
```

没有调任何统计库——WLS 的闭式解可以直接用累加器实现，复杂度和简单求和一样。a 的正负和大小反映了 21 日内对数价格的走势方向和强度。

### 2.4 年化收益率 × R²

```
annualized = exp(a × 250) − 1
```

A 股每年约 250 个交易日。a 是日均对数增长率，乘 250 再做 `exp` 还原，得到年化复合收益率。

然后是 R²（决定系数），衡量拟合线对价格走势的解释力：

```
R² = 1 − SSE_weighted / SST_weighted
```

其中 `SSE = Σ w_i × (y_i − ŷ_i)²`，`SST = Σ w_i × (y_i − ȳ)²`。

注意：**SST 中的均值 ȳ 必须用简单算术平均**（`Σy_i / n`），不能用加权均值。这个细节对齐了 `np.mean(y)` 的语义——如果用加权均值，R² 会系统性偏高，且对不同波动模式的序列敏感度不一致。

最后：

```
score = annualized × R²
```

这个设计的直觉非常清晰：**年化收益告诉你「涨得多快」，R² 告诉你「涨得稳不稳」**。一只年化 50% 但 R² 仅 0.1 的过山车 ETF，综合分不如年化 15% 但 R² 0.9 的斜坡型 ETF。

> 另外，R² **不做 [0, 1] 的 clamp**。允许负 R² 是一种刻意的设计——震荡下行的锯齿序列，拟合线斜率可能微弱为负但离散度极大，负 R² 会把它的 score 压成负值，从而被后续的 `MinScore=0` 过滤掉。如果强制 clamp 到 0，这条序列会「活下来」混进候选池。

## 三、池子设计——72 只 ETF × 18 个板块

选对池子比选对参数更重要。如果池子只有 10 只 ETF 且偏科技，策略本质上就是个科技动量策略，不具备轮动价值。

## 四、归一化的两条路

同一个原始动量分（`annualized × R²`），系统里存在两种后续处理方式。这是架构上最容易被误解的设计。

### 路径 A：sigmoid + R² 置信度（展示 / 人类可读）

```
normalized = sigmoid(score) × (0.5 + 0.5 × R²)
           = 100/(1 + e^{−2.5×score}) × confidence
```

sigmoid 把原始分（典型区间 -0.3 ~ 1.5）平滑映射到 0~100：score=0 → 50 分，score=1 → ~92 分。再乘以 R² 置信度乘子——R²=0 时打五折，R²=1 时满分还原。

这个分值的优点是"看起来合理"——不会出现 "score 0.3 给 90 分" 这种反直觉现象，适合展示在报告里给人看。

### 路径 B：rawQuantScore 线性映射（回测 / 决策计算）

```
mapped = 50 + raw_score × 50
```

直接从裸原始分线性映射，**不做 sigmoid 变换，不乘 R² 置信度**。用于回测引擎和 FinalAgent 的加权公式——保证排名与原始动量排序一致。

### 为什么需要两条路？

核心原因：sigmoid 的非线性特征会 **反转排名**。

sigmoid 在中间段最陡（0 附近），两端饱和。两只 ETF 原始分差 0.2，如果落在 mid-range（0.3 vs 0.5），sigmoid 后可能差 11 分；但如果落在饱和区（1.2 vs 1.4），sigmoid 后仅差 2 分。**同样的原始分差距，sigmoid 放大了中部差异、压缩了尾部差异**——这不是一个保序映射。

再加上 R² 置信度乘子——高 R² 低动量的标的可能超过低 R² 高动量的标的——排名的稳定性就彻底被破坏了。

因此：sigmoid 路径仅用于 **报告展示**（告诉用户"这只 ETF 综合评分 92 分"）；rawQuantScore 路径用于 **回测引擎和加权融合**（确保与策略的数学逻辑一致）。

## 五、回测验证

为了验证策略效果，系统内置了状态化的每日回测引擎，支持与聚宽裸动量口径的对齐对照。最近一期回测结果：

| 指标 | 数值 |
|------|------|
| 回测区间 | 2026-04-16 ~ 2026-06-15 |
| **累计净收益** | **+20.30%** |
| 胜率 | 61.54% |
| 简易 Sharpe | 3.01 |
| 最大回撤 | 5.39% |
| 盈亏比 | 1.88 |
| 基准（510300 buy&hold） | +3.65% |
| **Alpha** | **+16.66%** |

两个月内策略累计净值 1.2030，同期沪深 300 仅 +3.65%。单笔最大收益 +10.44%（通信 ETF 515880），单笔最大亏损 -5.23%。

不过需要诚实地指出：这个回测区间恰好是科技板块持续走强的环境——13 笔交易中有 10 笔落在科技板块。动量策略在趋势性行情中表现出色，但在震荡市中会频繁换仓被交易成本吃掉收益。**没有一个策略能适应所有市场，这是量化交易的第一课。**

## 六、这个策略的上限和边界

最后聊一下这个策略的边界——知道「不能做什么」比知道「能做什么」更重要。

### 动量策略的固有局限

1. **震荡市杀手**：没有趋势就没有动量。在横盘震荡的环境中，策略会频繁换仓，双边千分之 1.6 的摩擦成本会迅速侵蚀本金；
2. **V 型反转见顶即买**：动量策略不预测拐点——如果某 ETF 在暴涨顶点被选中，策略会照买不误，然后在接下来的回调中亏损；
3. **A 股的行业轮动速度**：有些年份（如 2022）板块切换极快，21 日的动量窗口追不上 5 日的轮动节奏。

### 工程层面的现实约束

- 数据源可靠性：腾讯财经 API 会加 WAF 验证，东方财富 push2his 偶尔返回空体——需要多级降级策略；
- 回测 ≠ 实盘：回测假设以收盘价成交，实盘有滑点和流动性问题；
- 单点故障：本系统跑在个人服务器上，没有灾备、没有运维、没有监控。

### 聚宽社区的起点

这个策略的数学骨架来自聚宽社区流传的 `get_etf_rank` 函数。聚宽是一个非常好的量化策略原型平台——你可以在上面用几分钟写一个策略、用几年数据做回测。但当你想把它部署成一个「每天早上自动跑信号、推送微信、结合消息面和宏观分析」的系统时，就需要自己动手做工程化了。

本项目的价值不在于发明了一个新策略，而在于：**把一个社区策略从原型平台里「拎出来」，做成了一套可部署、可扩展、可与其他 Agent 协作的工程系统**。
